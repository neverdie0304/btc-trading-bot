# 두 번째 전략 기획서: 5분봉 볼린저 밴드 평균회귀 전략

## 1. 전략 선택 근거

### 왜 평균회귀(Mean Reversion)인가?

**기존 전략과의 관계:**
현재 운용 중인 전략은 **모멘텀/추세추종** 스타일(EMA 정배열 + VWAP 돌파 + RSI 상승 확인)이다. 포트폴리오 분산의 핵심 원칙은 **수익 곡선의 상관관계를 최소화**하는 것이며, 모멘텀의 자연스러운 보완재는 평균회귀다.

**학술적 근거:**

| 연구 | 핵심 발견 |
|------|-----------|
| Jegadeesh (1990), *Journal of Finance* | 1~2주 이내 단기 수익률은 **음의 자기상관**(negative autocorrelation)을 보임 → 단기 반전 효과 존재 |
| Poterba & Summers (1988), *Journal of Financial Economics* | 자산 가격은 장기적으로 평균에 회귀하는 성질을 가짐 |
| Gatev, Goetzmann & Rouwenhorst (2006), *Review of Financial Studies* | 통계적 차익거래(평균회귀 기반)가 안정적 수익을 산출함을 실증 |
| Makarov & Schoar (2020), *Journal of Financial Economics* | 크립토 시장의 비효율성이 단기 평균회귀 기회를 제공 |
| Caporale & Plastun (2019) | 크립토 일중(intraday) 수익률에서 **과잉반응 후 반전 패턴** 확인 |

**크립토 5분봉에서의 적합성:**
- 크립토 시장은 24/7 운영으로 **유동성 변동 구간**이 존재하며, 이 구간에서 일시적 과잉 이탈 후 복귀 패턴이 빈번
- 5분봉은 노이즈와 시그널의 경계에 있어, 볼린저 밴드 이탈이 **통계적으로 유의미한 과잉 이동**을 포착하기에 적합
- 모멘텀 전략이 **추세 구간**에서 수익을 내는 반면, 평균회귀 전략은 **레인지/횡보 구간**에서 수익을 내므로 시장 국면별 보완 효과가 큼

## 2. 전략 개요

> **한 줄 요약:** 볼린저 밴드 ±2σ 이탈 후 밴드 내부로 복귀하는 가격의 평균회귀 성질을 이용하여, 과매도/과매수 극단에서 역추세 진입하고 중심선(SMA20) 방향으로 수익을 실현한다.

### 작동 원리

```
가격 분포 가정: 5분봉 종가는 단기적으로 SMA(20) 주위에 정규분포에 가깝게 분포
통계적 근거: 볼린저 밴드 ±2σ 바깥 = 약 5%의 극단 영역
핵심 가설: 극단 이탈은 일시적 과잉반응이며, 높은 확률로 밴드 내부로 복귀한다
수익 원천: 과잉반응(overreaction) → 복귀(reversion) 사이의 가격 차이
```

**기존 모멘텀 전략과의 차이점:**

| 항목 | 기존 전략 (모멘텀) | 신규 전략 (평균회귀) |
|------|-------------------|---------------------|
| 시장 국면 | 추세(Trending) | 횡보/레인지(Ranging) |
| 진입 방향 | 추세 방향 순행 | 추세 반대 역행 |
| 승률 | 중간 (55~65%) | 높음 (65~75%) |
| 손익비 | 높음 (1:1.5~2.0) | 낮음 (1:0.8~1.2) |
| 보유 시간 | 중간 (25~45분) | 짧음 (10~25분) |
| 상관관계 | - | 기존 대비 낮은 상관 기대 |

## 3. 신호 생성 로직

### 사용 지표

```
1. Bollinger Bands (20, 2.0)
   - Middle: SMA(20)
   - Upper: SMA(20) + 2.0 × σ(20)
   - Lower: SMA(20) - 2.0 × σ(20)

2. %B (Percent B)
   - %B = (close - lower_band) / (upper_band - lower_band)
   - %B < 0: 하단 밴드 이탈  /  %B > 1: 상단 밴드 이탈

3. Bandwidth (밴드 폭)
   - BW = (upper - lower) / middle
   - 밴드 스퀴즈 감지용

4. RSI(7) — 단기 과매도/과매수 확인 (기존 전략의 RSI(14)와 기간 차별화)

5. Stochastic RSI(7, 7, 3, 3) — RSI의 상대 위치로 반전 타이밍 포착

6. ATR(14) — 변동성 기반 SL/TP 동적 산출

7. Volume MA(20) 대비 비율 — 클라이맥스 거래량 감지
```

### LONG 진입 조건 (5개 모두 충족)

```
1. [밴드 이탈 후 복귀]
   prev_close < lower_band AND close > lower_band
   (= 이전 봉에서 하단 밴드 이탈 → 현재 봉에서 밴드 안으로 복귀)

2. [과매도 확인]
   RSI(7) < 25 (1봉 전 기준) AND RSI(7) > RSI(7)[1봉 전] (RSI 반등 시작)

3. [Stochastic RSI 반전]
   StochRSI_K > StochRSI_D AND StochRSI_K[1봉 전] <= StochRSI_D[1봉 전]
   (= 골든크로스 발생)

4. [거래량 클라이맥스]
   volume > volume_ma20 × 1.5
   (= 과매도 구간에서의 투매 거래량 → 셀링 클라이맥스 감지)

5. [밴드폭 필터]
   bandwidth > bandwidth_SMA(50) × 0.8
   (= 밴드가 너무 좁으면 스퀴즈 상태 → 평균회귀가 아닌 돌파 가능성이 높으므로 제외)
```

### SHORT 진입 조건 (5개 모두 충족)

```
1. [밴드 이탈 후 복귀]
   prev_close > upper_band AND close < upper_band

2. [과매수 확인]
   RSI(7) > 75 (1봉 전 기준) AND RSI(7) < RSI(7)[1봉 전] (RSI 하락 시작)

3. [Stochastic RSI 반전]
   StochRSI_K < StochRSI_D AND StochRSI_K[1봉 전] >= StochRSI_D[1봉 전]
   (= 데드크로스 발생)

4. [거래량 클라이맥스]
   volume > volume_ma20 × 1.5
   (= 과매수 구간에서의 FOMO 거래량 → 바잉 클라이맥스 감지)

5. [밴드폭 필터]
   bandwidth > bandwidth_SMA(50) × 0.8
```

### 필터 조건 (진입 차단)

```
- 기존 전략과 공유: FOMC/CPI/NFP 블랙아웃, 주말 필터
- ADX(14) > 30 → 강한 추세 구간에서는 평균회귀 진입 금지
  (= 핵심 필터. 추세가 강하면 밴드 이탈이 "과잉반응"이 아닌 "추세 시작"일 수 있음)
- 밴드폭 하위 20% 구간 → 스퀴즈 상태에서 진입 금지
- 최근 연속 4패 시 → 다음 24봉(2시간) 쿨다운
  (평균회귀 실패 = 추세 전환 가능성 → 더 긴 쿨다운 부여)
```

## 4. 손익비 설계

### 핵심 원칙

평균회귀 전략은 **높은 승률 + 낮은 R:R**로 설계한다. 무리하게 R:R을 높이면 밴드 중심까지 도달하기 전에 가격이 다시 이탈하여 승률이 급락한다.

### SL/TP 산출

```python
# LONG 진입 시
entry_price = close  # 봉 종가 확정 후 진입
SL = entry_price - 0.8 × ATR(14)     # 타이트한 손절
TP1 = SMA(20)                         # 1차 목표: 밴드 중심선 (50% 물량)
TP2 = entry_price + 1.2 × ATR(14)    # 2차 목표: 추가 수익 (50% 물량)

# SHORT 진입 시
entry_price = close
SL = entry_price + 0.8 × ATR(14)
TP1 = SMA(20)                         # 1차 목표: 밴드 중심선 (50% 물량)
TP2 = entry_price - 1.2 × ATR(14)    # 2차 목표 (50% 물량)
```

### 분할 청산 로직

```
진입 직후:
  - 물량의 50% → TP1 (밴드 중심선)에서 청산
  - 물량의 50% → TP2에서 청산 또는 트레일링 스탑 적용

TP1 도달 시:
  - 잔여 50% 물량의 SL을 진입가(Break-Even)로 이동
  - TP2까지 보유하되, SMA(20) 반대편으로 종가 확정 시 즉시 청산

목표 R:R (가중 평균):
  - TP1: 약 0.6~0.9R (밴드 중심까지의 거리 / SL 거리)
  - TP2: 약 1.5R
  - 가중 평균: (0.75R × 0.5) + (1.5R × 0.5) ≈ 1.1R
  - 승률 68%+ 기대 → 기대값 양수
```

### 기대값 계산

```
기대값 = (승률 × 평균이익) - (패률 × 평균손실)
       = (0.68 × 1.1R) - (0.32 × 1.0R)
       = 0.748R - 0.32R
       = +0.428R (거래당)
```

## 5. 레버리지 최적화

### Kelly Criterion 기반 산출

```
Kelly 공식: f* = (p × b - q) / b

여기서:
  p = 승률 = 0.68
  q = 패률 = 0.32
  b = 평균이익 / 평균손실 = 1.1R / 1.0R = 1.1

f* = (0.68 × 1.1 - 0.32) / 1.1
   = (0.748 - 0.32) / 1.1
   = 0.428 / 1.1
   = 0.389 (38.9%)
```

### 실전 적용 — Fractional Kelly

```
Full Kelly는 이론적 최적이지만, 현실에서는 파라미터 추정 오차와
블랙스완 리스크를 감안하여 1/4 ~ 1/2 Kelly를 사용한다.

Half Kelly:    f = 0.389 / 2 = 0.195 (19.5%)
Quarter Kelly: f = 0.389 / 4 = 0.097 (9.7%)

리스크 비율(risk_per_trade) = 1.5% 기준:
  position_size = (capital × 0.015) / sl_distance

  SL 거리 = 0.8 × ATR(14)
  BTC 가격 ~$60,000 기준, ATR(14) ≈ $120
  SL 거리 ≈ $96

  자본 $10,000:
  position_size_notional = ($10,000 × 0.015) / $96 × $60,000
                         ≈ $150 / $96 × $60,000
                         ≈ 0.00156 BTC ≈ $93.75 notional

  실효 레버리지 = $93.75 / $10,000 ≈ 0.94x (매우 보수적)
```

### 레버리지 제안

```
┌─────────────────────────────────────────────────┐
│  제안: 계좌 레버리지 10x 설정 / 실효 레버리지 3~5x  │
└─────────────────────────────────────────────────┘

근거:
1. 평균회귀 전략은 모멘텀 대비 체결 빈도가 높고 보유시간이 짧음
   → 같은 레버리지에서도 리스크 노출 시간이 적음

2. 그러나 평균회귀 실패(= 추세 전환) 시 손실이 급격히 확대될 수 있음
   → 모멘텀 전략(20x)보다 낮은 레버리지가 안전

3. 두 전략 합산 포트폴리오 관점:
   - 전략 1 (모멘텀): 20x 레버리지, risk 1.5%/trade
   - 전략 2 (평균회귀): 10x 레버리지, risk 1.0%/trade
   - 합산 최대 동시 리스크: 2.5% (상관관계 낮으므로 허용 가능)

4. Quarter Kelly (9.7%) 기준으로 역산:
   risk_per_trade = 1.0%
   position_size = capital × 0.01 / sl_distance
   실효 레버리지 ≈ 3~5x (ATR에 따라 변동)
   계좌 레버리지 10x → 마진 여유 충분
```

## 8. 리스크 및 한계점

### 전략 붕괴 시나리오

| 시나리오 | 원인 | 영향 | 대응 방안 |
|----------|------|------|-----------|
| **추세 지속** | 강한 단방향 움직임에서 밴드 이탈이 "과잉반응"이 아닌 추세 시작 | 연속 손절, MDD 확대 | ADX 필터 강화 (>25 시 진입 금지), 쿨다운 연장 |
| **변동성 급변** | 블랙스완 이벤트로 ATR 급등, SL이 멀어지면서 포지션 사이즈 과소 | 수익 기회 축소 | ATR 상한 캡 설정 (median × 3.0 초과 시 진입 금지) |
| **스프레드 확대** | 유동성 급감 시 실제 슬리피지가 가정치 초과 | 비용 증가로 기대값 음전환 | 유동성 필터 추가 (스프레드 모니터링), 진입 수량 축소 |
| **시장 구조 변화** | 크립토 시장 성숙에 따른 비효율 감소 | 평균회귀 기회 자체가 줄어듦 | 파라미터 정기 재최적화 (분기별), 성과 모니터링 후 전략 교체 |
| **동시 손실** | 두 전략이 동시에 손실 구간 진입 (급변장) | 합산 MDD 예상 초과 | 포트폴리오 레벨 kill switch: 합산 일일 손실 -3% 시 전체 중지 |

### 구조적 한계

```
1. 평균회귀는 "대부분의 시간" 작동하지만,
   "작동하지 않는 순간"의 손실이 클 수 있다 (fat tail risk)
   → 반드시 SL 엄수, 절대 SL 없이 버티기 금지

2. 5분봉 평균회귀는 거래 빈도가 높아 수수료 부담이 큼
   → Binance VIP 등급 상향 또는 maker 주문 비율 최대화

3. 파라미터 과적합 리스크
   → Walk-forward validation 필수, 조건 수 최소화 원칙 준수
```
