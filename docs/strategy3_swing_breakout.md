## 1. 전략 선택 근거

### 왜 구조 브레이크아웃(Structural Breakout)인가?

**기존 포트폴리오 구성:**

| # | 전략 | 타임프레임 | 스타일 | 보유 시간 | 시장 국면 |
|---|------|-----------|--------|-----------|-----------|
| 1 | 모멘텀 스캘핑 | 5분봉 | 추세추종 | 25~45분 | 추세 |
| 2 | 볼린저 평균회귀 | 5분봉 | 역추세 | 10~25분 | 횡보 |
| 3 | **구조 브레이크아웃** | **1시간봉** | **스윙** | **4시간~3일** | **국면 전환** |

세 번째 전략은 **타임프레임**, **보유 기간**, **수익 원천** 세 축 모두에서 기존 전략과 차별화되어야 한다. 1시간봉 구조 브레이크아웃은 **가격 구조(Price Structure)의 전환점**에서 진입하여 **중기 추세의 초입**을 포착하므로, 5분봉 단타 전략들과 시간적·논리적 상관관계가 최소화된다.

**학술적 근거:**

| 연구 | 핵심 발견 |
|------|-----------|
| Donchian (1960), Curtis Faith (2007) *Way of the Turtle* | N일 고/저점 돌파 전략(Donchian Breakout)이 수십 년간 다양한 시장에서 양의 기대값을 산출. 터틀 트레이딩의 핵심 원리 |
| Moskowitz, Ooi & Pedersen (2012), *Journal of Financial Economics* | **Time-Series Momentum(TSMOM)**: 1~12개월 수익률이 양의 자기상관을 보임. 선물 시장에서 가장 강력한 이상현상(anomaly) 중 하나 |
| Asness, Moskowitz & Pedersen (2013), *Journal of Finance* | 모멘텀 효과는 자산군, 시장, 시간대를 초월하여 존재. 크립토 포함 대체자산에도 적용 |
| Hurst, Ooi & Pedersen (2017), *Journal of Portfolio Management* | 추세추종이 위기 상황에서 **볼록한 수익 구조(convexity)**를 제공 — 꼬리 위험 헤지 효과 |
| Liu & Tsyvinski (2021), *Review of Financial Studies* | 크립토 자산의 수익률은 **모멘텀 팩터**로 유의미하게 설명됨. 전통 자산 팩터(시장, 규모, 가치)로는 설명 불가 |
| Bianchi (2020), *Journal of Financial Economics* | 크립토 시장에서 중기(1주~1개월) 추세추종이 단기/장기 대비 가장 안정적인 수익 산출 |

**1시간봉 스윙에서의 적합성:**
- 5분봉 노이즈가 제거되어 **구조적 고점/저점(swing high/low)**이 명확하게 식별됨
- 보유 기간 4시간~3일은 크립토의 **중기 모멘텀 사이클**과 일치
- 1시간봉은 기관/고래의 **구조적 매집/분배 패턴**이 반영되는 최소 단위
- 브레이크아웃 전략은 **추세 전환 초기**에 진입하므로, 기존 5분봉 추세추종(추세 중반 진입)과 진입 시점이 다름

## 2. 전략 개요

> **한 줄 요약:** 4시간봉 추세 방향을 필터로 삼아, 1시간봉에서 Donchian Channel(20봉) 돌파와 시장 구조 전환(Higher High/Lower Low)이 동시에 발생하는 지점에서 스윙 진입하고, ATR 기반 트레일링 스탑으로 추세를 최대한 추종한다.

### 작동 원리

```
핵심 논리: "가격이 N봉 고점을 돌파하면, 새로운 추세가 시작될 확률이 높다"
          + "상위 타임프레임(4H)의 추세 방향으로만 진입하면 승률이 크게 향상된다"

수익 원천: 중기 추세(4시간~수 일)의 초기 진입 → 추세 지속 구간에서의 마크업/마크다운 수익
핵심 엣지: 구조 전환 + 상위 TF 확인의 이중 필터로 거짓 돌파(false breakout)를 걸러냄
손익 구조: 낮은 승률(40~50%) + 높은 R:R(1:2.5~4.0) → 소수의 큰 수익 거래가 전체를 주도
```

**세 전략의 수익 곡선 보완 구조:**

```
시장 국면        전략 1 (5m 모멘텀)  전략 2 (5m MR)  전략 3 (1H 스윙)
─────────────  ──────────────────  ──────────────  ───────────────
강한 추세       ★★★ (주력)         ✗ (필터링)      ★★★ (초기 포착)
약한 추세       ★★ (보통)          ★ (간헐)        ★★ (보통)
횡보/레인지     ✗ (신호 없음)       ★★★ (주력)     ✗ (거짓 돌파)
국면 전환       ✗ (늦은 진입)       ★ (반전)        ★★★ (구조 전환 포착)
급변/블랙스완   ✗ (SL)             ✗ (SL)          ★★ (볼록성 수혜)
```

## 3. 신호 생성 로직

### 사용 지표

```
[ 1시간봉 지표 ]
1. Donchian Channel (20)
   - Upper: 최근 20봉 최고가
   - Lower: 최근 20봉 최저가
   - Middle: (Upper + Lower) / 2

2. 시장 구조 (Swing High/Low)
   - Swing High: 좌우 각 3봉보다 높은 고점 (lookback=3)
   - Swing Low: 좌우 각 3봉보다 낮은 저점 (lookback=3)
   - Higher High (HH) / Higher Low (HL) → 상승 구조
   - Lower High (LH) / Lower Low (LL) → 하락 구조

3. ATR(14) — 변동성 기반 SL/TP, 포지션 사이징

4. Volume MA(20) 대비 비율 — 돌파 시 거래량 확인

5. ADX(14) — 추세 강도 확인

[ 4시간봉 지표 — 상위 TF 필터 ]
6. EMA(21) on 4H — 중기 추세 방향
7. EMA(50) on 4H — 장기 추세 방향
8. 4H 봉의 위치: close > EMA21_4H > EMA50_4H → 상승 추세
```

### LONG 진입 조건 (5개 모두 충족)

```
1. [상위 TF 추세 확인]
   4H close > 4H EMA(21) > 4H EMA(50)
   (= 4시간봉 기준 상승 추세 확인 — 추세 방향으로만 진입)

2. [Donchian 상단 돌파]
   close > Donchian_Upper(20)
   (= 최근 20봉(20시간) 최고가를 종가 기준으로 돌파)

3. [시장 구조 전환]
   직전 Swing Low가 이전 Swing Low보다 높음 (Higher Low 형성)
   AND 현재 돌파가 직전 Swing High를 상회 (Higher High 형성)
   (= 상승 구조 전환 확인 — HL → HH 시퀀스)

4. [거래량 동반]
   volume > volume_ma20 × 1.3
   (= 구조적 돌파에 거래량이 실려야 유효한 돌파)

5. [최소 변동성]
   ADX(14) > 20
   (= 추세가 형성되기 시작하는 최소 강도)
```

### SHORT 진입 조건 (5개 모두 충족)

```
1. [상위 TF 추세 확인]
   4H close < 4H EMA(21) < 4H EMA(50)

2. [Donchian 하단 이탈]
   close < Donchian_Lower(20)

3. [시장 구조 전환]
   직전 Swing High가 이전 Swing High보다 낮음 (Lower High 형성)
   AND 현재 이탈이 직전 Swing Low를 하회 (Lower Low 형성)
   (= 하락 구조 전환 확인 — LH → LL 시퀀스)

4. [거래량 동반]
   volume > volume_ma20 × 1.3

5. [최소 변동성]
   ADX(14) > 20
```

### 필터 조건 (진입 차단)

```
- FOMC/CPI/NFP 블랙아웃 전후 2시간 (1H봉 기준 더 넓은 버퍼)
- 주말 필터: 토요일 00:00 ~ 일요일 12:00 UTC (스윙이므로 더 넓게)
- 4H EMA(21)과 EMA(50) 간격 < ATR_4H × 0.3 → 추세 불명확, 진입 금지
- Donchian 채널 폭(Upper - Lower) < ATR(14) × 3.0 → 수렴 구간, 진입 금지
  (= 채널이 지나치게 좁으면 거짓 돌파 확률 급증)
- 최근 연속 3패 시 → 다음 12봉(12시간) 쿨다운
- 일일 최대 진입 횟수: 2회 (스윙 전략은 과잉 매매 방지가 핵심)
```

## 4. 손익비 설계

### 핵심 원칙

브레이크아웃 스윙 전략은 **낮은 승률 + 높은 R:R**로 설계한다. 거짓 돌파에서의 소규모 손절을 반복하되, 진짜 추세를 잡았을 때 큰 수익으로 만회한다. **트레일링 스탑이 핵심** — 익절 목표를 고정하지 않고 추세가 지속되는 한 보유한다.

### SL/TP 산출

```python
# LONG 진입 시
entry_price = close  # 1H 봉 종가 확정 후 진입

# 손절: Donchian Middle 또는 ATR 기반 중 넓은 것
SL_option_1 = Donchian_Middle(20)                  # 채널 중심선
SL_option_2 = entry_price - 1.5 × ATR(14)          # ATR 기반
SL = max(SL_option_1, SL_option_2)                  # 더 가까운(타이트한) 쪽 선택
# 단, SL 거리가 ATR × 2.5를 초과하면 진입 포기 (리스크 과다)

# 1차 익절 (부분 청산)
TP1 = entry_price + 2.0 × ATR(14)                  # 30% 물량 청산

# 최종 익절 (트레일링 스탑으로 대체)
TP 고정 목표 없음 — Chandelier Exit 트레일링 스탑 사용

# SHORT 진입 시
SL_option_1 = Donchian_Middle(20)
SL_option_2 = entry_price + 1.5 × ATR(14)
SL = min(SL_option_1, SL_option_2)

TP1 = entry_price - 2.0 × ATR(14)                  # 30% 물량 청산
```

### Chandelier Exit 트레일링 스탑

```
기존 5분봉 전략의 R배수 기반 트레일링과 차별화하여,
스윙에 적합한 Chandelier Exit을 사용한다.

LONG:
  trailing_SL = highest_high(N봉) - 3.0 × ATR(14)
  N = 진입 이후 경과 봉 수 (최소 1)
  → 가격이 상승할수록 trailing_SL이 올라가며, 되돌림 시 청산

SHORT:
  trailing_SL = lowest_low(N봉) + 3.0 × ATR(14)
  → 가격이 하락할수록 trailing_SL이 내려가며, 반등 시 청산

ATR 배수 3.0의 근거:
  - 1시간봉은 5분봉 대비 노이즈가 크므로 넓은 여유 필요
  - ATR × 3.0은 약 99.7% 신뢰구간에 해당하는 정상 되돌림 흡수
  - 너무 타이트하면 정상 조정에 조기 청산 → R:R 급락
```

### 분할 청산 로직

```
단계 1: 진입 직후
  - 전체 물량 보유, SL = 초기 SL

단계 2: TP1 도달 (2.0R)
  - 30% 물량 청산 (수익 확보)
  - 잔여 70% 물량의 SL을 진입가(Break-Even)로 이동

단계 3: 3.0R 도달
  - Chandelier Exit 트레일링 스탑 활성화
  - 이후 트레일링 SL에 히트할 때까지 보유

단계 4: 청산
  - 트레일링 SL 히트 시 잔여 70% 전량 청산
  - 또는 4H EMA(21) 하회(LONG) / 상회(SHORT) 종가 확정 시 전량 청산

목표 R:R 분포:
  - 손절 거래: -1.0R (100% 물량)
  - TP1 후 BE 청산: +0.6R (가중 평균: 2.0R×0.3 + 0R×0.7)
  - 풀 트레일링 수익: +2.5~5.0R (가중 평균: 2.0R×0.3 + 3.0~6.0R×0.7)
  - 가중 평균 R:R ≈ 1:2.7
```

### 기대값 계산

```
승률 분포 가정:
  - 즉시 손절 (-1.0R): 45%
  - TP1 후 BE 청산 (+0.6R): 20%
  - 트레일링 수익 (+3.5R 평균): 35%

기대값 = (0.45 × -1.0R) + (0.20 × +0.6R) + (0.35 × +3.5R)
       = -0.45R + 0.12R + 1.225R
       = +0.895R (거래당)

→ 단일 거래 기대값이 높지만 거래 빈도가 낮음 (주 3~6회)
→ 월간 기대 수익: 0.895R × 4.5회/주 × 4주 ≈ 16.1R
```

## 5. 레버리지 최적화

### Kelly Criterion 기반 산출

```
승률과 R:R이 비대칭 분포이므로, 일반화된 Kelly 공식 사용:

f* = Σ(p_i × r_i) / Σ(p_i × r_i²)  [간략화]

또는 보수적으로 단순 Kelly:
  유효 승률 p = 0.55 (TP1 이상 도달 확률 = 20% + 35%)
  유효 패율 q = 0.45
  유효 b = 평균이익 / 평균손실
         = [(0.20/0.55 × 0.6R) + (0.35/0.55 × 3.5R)] / 1.0R
         = [0.218R + 2.227R] / 1.0R
         = 2.445

f* = (p × b - q) / b
   = (0.55 × 2.445 - 0.45) / 2.445
   = (1.345 - 0.45) / 2.445
   = 0.895 / 2.445
   = 0.366 (36.6%)
```

### 실전 적용 — Fractional Kelly

```
스윙 전략은 보유 기간이 길어 블랙스완 노출이 크므로 1/4 Kelly 적용.

Quarter Kelly: f = 0.366 / 4 = 0.092 (9.2%)

그러나 스윙 전략의 SL 거리가 넓으므로 (1.5 × ATR),
risk_per_trade를 낮게 잡아도 실효 레버리지가 자연스럽게 제한됨.
```

### 레버리지 제안

```
┌──────────────────────────────────────────────────────┐
│  제안: 계좌 레버리지 5x 설정 / 실효 레버리지 1.5~3x    │
└──────────────────────────────────────────────────────┘

근거:
1. 1시간봉 스윙은 보유 시간이 4시간~3일
   → 야간/주말 갭 리스크, 예상치 못한 뉴스 리스크 노출
   → 5분봉 전략(20x, 10x) 대비 낮은 레버리지 필수

2. SL 거리가 넓음 (1.5 × ATR_1H ≈ $300~500 on BTC)
   → risk_per_trade = 0.8%로도 적정 포지션 사이즈 확보

3. BTC $60,000 기준 계산:
   ATR(14)_1H ≈ $250
   SL 거리 = 1.5 × $250 = $375
   자본 $10,000, risk = 0.8%:
   position_size = ($10,000 × 0.008) / $375 = 0.0213 BTC
   notional = 0.0213 × $60,000 = $1,280
   실효 레버리지 = $1,280 / $10,000 = 1.28x

4. 세 전략 합산 포트폴리오 최대 동시 리스크:
   - 전략 1 (5m 모멘텀):    risk 1.5% × 20x leverage
   - 전략 2 (5m 평균회귀):  risk 1.0% × 10x leverage
   - 전략 3 (1H 스윙):      risk 0.8% × 5x leverage
   - 합산 최대 동시 리스크: 3.3%
   - 단, 전략 1&2는 동시 포지션 가능성 낮음 (추세↔횡보)
   - 실질 동시 리스크: ~2.3% (관리 가능)
```

### 검증 기간

```
In-Sample (파라미터 최적화):  2022-01-01 ~ 2024-06-30 (30개월)
Out-of-Sample (검증):         2024-07-01 ~ 2025-12-31 (18개월)
Walk-Forward (추가 검증):     6개월 단위 rolling window × 6회

총 데이터: 약 35,000봉 (1시간봉 × 48개월)
4시간봉 데이터: 약 8,700봉 (상위 TF 필터용)

※ 스윙 전략은 거래 빈도가 낮으므로 충분한 거래 표본(최소 200회)을 확보하기 위해
  5분봉 전략(18개월) 대비 긴 IS 기간(30개월) 적용
```

### 비용 가정

```
수수료:   편도 0.04% (Binance Futures maker/taker 평균)
슬리피지: 편도 0.02% (1H봉은 급변 진입이 적어 슬리피지 낮게 가정)
총 왕복:  (0.04% + 0.02%) × 2 = 0.12%

분할 청산 고려:
  - TP1 부분 청산 + 트레일링 청산 = 진입 1회 + 청산 2회
  - 총 비용 = 약 0.18%

펀딩비 (스윙 전략 고유 비용):
  - Binance 영구선물 펀딩비: 8시간마다 ~0.01% (변동)
  - 평균 보유 시간 24시간 가정 → 펀딩비 3회 ≈ 0.03%
  - 총 거래 비용: 0.18% + 0.03% = 0.21%
```

### 최적화 파라미터 범위

```
Donchian 기간:         [15, 20, 25, 30]
Swing lookback:        [2, 3, 4, 5]
4H EMA fast:           [14, 21, 30]
4H EMA slow:           [40, 50, 60]
SL ATR 배수:           [1.2, 1.5, 2.0]
TP1 ATR 배수:          [1.5, 2.0, 2.5]
Chandelier ATR 배수:   [2.5, 3.0, 3.5]
ADX 최소:              [15, 20, 25]
Volume 배수:           [1.2, 1.3, 1.5]

총 조합: 4 × 4 × 3 × 3 × 3 × 3 × 3 × 3 × 3 = 104,976개
→ Random Search (5,000회) 또는 Bayesian Optimization 적용
→ 과적합 방지: OOS 성과가 IS의 50% 이상 유지 + 파라미터 안정성 분석
```

### 파라미터 안정성 분석 (Robustness Test)

```
최적 파라미터 기준으로 ±1 단위 변경 시 성과 변동 확인:
  - Sharpe 변동 < 20% → 안정적
  - Sharpe 변동 > 40% → 과적합 의심 → 해당 파라미터 제외

예: Donchian 기간 최적값 = 20
  → 18, 19, 21, 22에서도 유사 Sharpe 유지 확인
```

## 8. 리스크 및 한계점

### 전략 붕괴 시나리오

| 시나리오 | 원인 | 영향 | 대응 방안 |
|----------|------|------|-----------|
| **연속 거짓 돌파** | 가격이 Donchian 상단 돌파 후 즉시 반전을 반복 (Whipsaw) | 연속 -1R 손절 누적, 승률 30% 이하로 추락 | Donchian 기간 연장 (20→25), 진입 확인 봉 추가 (돌파 후 1봉 대기) |
| **추세 없는 장기 횡보** | BTC가 수 주간 좁은 레인지에 갇힘 | 신호 자체가 줄어들고, 발생해도 거짓 돌파 | ADX < 20 필터 작동, 자연스럽게 거래 감소 → 손실 제한 |
| **갭/급변 이벤트** | 규제 뉴스, 거래소 해킹 등으로 수 분 내 10%+ 급변 | SL 슬리피지 확대, 예상 -1R 초과 손실 | 최대 손실 캡 설정 (-2R 초과 시 즉시 시장가 청산), 낮은 레버리지(5x)가 완충 |
| **펀딩비 역풍** | 장기 보유 시 펀딩비가 수익을 잠식 | 수익성 저하 | 펀딩비 모니터링: 누적 펀딩비 > 0.1% 시 조기 청산 고려, 극단적 펀딩비(>0.05%/8h) 시 진입 금지 |
| **상위 TF 신호 지연** | 4H EMA가 추세 전환을 늦게 반영하여 역추세 진입 | 추세 초입에서 손절 | 4H EMA 기간 단축 고려 (21→14), 또는 4H 구조 전환도 함께 확인 |
| **세 전략 동시 손실** | 급변장에서 모든 전략이 동시에 SL 히트 | 포트폴리오 레벨 -3.3% 일일 손실 | 포트폴리오 kill switch: 합산 일일 손실 -4% 시 전체 봇 정지, 수동 리뷰 후 재가동 |

### 구조적 한계

```
1. 거래 빈도가 낮다 (주 3~6회)
   → 통계적 유의성 확보에 오랜 기간 필요 (최소 6개월 라이브 운용)
   → 단기 성과에 일희일비하지 않는 심리적 내성 필요

2. 보유 기간이 길어 심리적 부담이 큼
   → 미실현 손실 구간에서 SL 이동/조기 청산 유혹
   → 반드시 자동화 실행, 수동 개입 금지 원칙

3. 거짓 돌파(False Breakout) 비율이 50%+
   → 전략의 본질적 한계. 높은 R:R로 보상받는 구조
   → 연속 5~7패가 통계적으로 정상 범위 (심리적 대비 필요)

4. 4시간봉 필터로 인한 지연
   → 추세 전환 초기의 첫 1~2회 돌파를 놓칠 수 있음
   → 이는 의도된 트레이드오프: 승률 향상을 위한 진입 기회 희생

5. 크립토 영구선물의 펀딩비
   → 전통 선물에는 없는 추가 비용
   → 백테스트에 반드시 펀딩비 반영 필수
```
